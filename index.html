Papers 연결하기

1. 구글스프레드시트 만들기

  1. 템플릿에 접속합니다.
  2. 파일 사본만들기 버튼을 클릭해서 자신의 구글 스프레드 시트에 사본을 저장합니다.
    (이하 "시트"로 지칭합니다. )
  3. 시트의 탭의 제목을 변경합니다.
  4. 시트에서 하얀색 부분을 삭제하고, Papers에 필요한 정보를 채워넣습니다.

2. 웹에 게시하기

  1. 시트에서 파일 웹에게시를 클릭합니다.
  2. 파란색 게시 버튼을 클릭하세요.
  3. 공유되어 나온 링크 URL을 복사해서 아래의 입력폼에 넣어주세요.

3. Papers URL 만들기.

시트에서 복사된 공유링크를 넣으시고, Papers 주소받기 버튼을 누르세요!

4. 활용팁!

  • Papers는 Github와 구글스프레드시트가 망하지 않는한 어떤 기기든 접근이 가능합니다.
  • 루트경로에 http:// 로 시작하는 경로를 심으면 외부에서도 볼수 있습니다.
  • 시트를 "웹에게시" 할꺼면 그냥 전체공유 걸어버리면 기획자에게 검수를 맡길수 있습니다!
  • Papers의 경로는 http://dstyle0210.github.io/papers/dist/papers.html?papers=스프레드시트아이디로 구성되어있습니다.

"Papers" 란?

구글 스프레드시트의 정보를 기반으로 웹퍼블리싱 진행상태를 확인합니다.

  • 웹사이트 구축에 필요한 전체구조을 파악할수 있습니다.
  • 현재 진행중인 웹퍼블리싱 상황을 알수 있습니다.
  • 링크를 통하여 바로 연결이 가능합니다.
  • 기획자와 협업한다면, 개발자에게 공지하지 않아도 업무진행상황을 알 수 있습니다.
  • 산출물로 재정리를 하지 않아도 됩니다.

Papers 상세 도움말(QnA)

  • 각 항목 소개

    카테고리
    실제 웹사이트의 메인섹션 구분입니다. 메인 및 풋터 , 공통 부분들로 나눌수 있습니다.
    카테고리는 구분인자 일뿐으로, 진행상태에 대해서는 넣지 마세요.
    1뎁스 , 2뎁스 , 3뎁스
    웹사이트 세부구조 입니다. 웹사이트의 구조에 맞추어 넣어주세요. 2뎁스는 1뎁스가 필수요소입니다. (물론 3뎁스는 1,2뎁스가 필수입니다.)
    최종경로
    최종경로는 링크값이 되기 위한것입니다.(루트폴더 + 중간폴더 + 세부폴더 + 파일명 + 확장자)가 최종경로가 됩니다.
    최종경로는 Papers 안에서 링크경로로 사용됩니다.
    시작일 , 종료일
    업무 시작일과 , 종료일 입니다. 진행상황에서 종료일이 보여집니다.
    종료일이 지났는데도, "상태"에서 퍼블완료가 되어있지 않으면 빨간색 아이콘으로 변경됩니다.
    업무자 , 검수자
    업무자는 퍼블리셔 이름 , 검수자는 스스로 할수도 있고, 기획자에게 요청할수도 있습니다.
    비고
    비고는 부트스트랩기준 sm이하로 내려가면, 안보이게 됩니다.

  • 1행이 없어요.

    구글스프레드시트의 정보를 정리하기 위해서 JSON key명이 들어가 있습니다.(수정 및 삭제하지 마세요) 시트에서는 도움 안되니 숨기기 처리해주세요.
